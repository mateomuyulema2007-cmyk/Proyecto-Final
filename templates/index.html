Variables y Tipos

Una variable guarda informacion para que el programa pueda recordarla, modificarla y usarla en calculos o decisiones.

Salir

Informacion ampliada

Las variables son espacios de memoria identificados con un nombre. En programacion basica se usan para almacenar datos como notas, edades, precios, nombres, respuestas y estados logicos. Elegir buenos nombres ayuda a que el algoritmo sea entendible.

Los tipos de datos indican que clase de valor se guarda. Los mas comunes son enteros, decimales, cadenas y booleanos. Cada tipo permite operaciones distintas: con numeros se calcula, con cadenas se construyen mensajes y con booleanos se toman decisiones.

TipoUsoEjemplo
intCantidades enterasedad = 19
floatValores con decimalespromedio = 8.75

boolVerdadero o falsoaprobado = True

Ejemplo aplicado

Problema: registrar informacion basica de un estudiante y mostrar un resumen.

nombre = "Mateo"
edad = 18
promedio = 8.6
aprobado = promedio >= 7

print(nombre, edad, promedio, aprobado)
Entrada: nombre, edad y promedio.
Proceso: comparar si el promedio es suficiente.
Salida: resumen del estudiante y estado.

Condicionales

Los condicionales permiten que un programa tome decisiones segun una condicion.

Salir

Informacion ampliada

Una estructura condicional evalua una expresion logica. Si la condicion es verdadera, ejecuta un bloque de instrucciones; si es falsa, puede ejecutar otro bloque. Por eso los condicionales son fundamentales para validar entradas, clasificar datos y generar retroalimentacion personalizada.

En Python se usan if, elif y else. En C++ y Java se usa la misma idea con llaves. Lo importante no es memorizar la sintaxis, sino entender la pregunta logica que el programa esta haciendo.

if promedio >= 7:
  estado = "Aprobado"
elif promedio >= 5:
  estado = "Supletorio"
else:
  estado = "Reprobado"

Bucles y Ciclos

Un ciclo repite instrucciones mientras se cumpla una condicion o mientras existan elementos por recorrer.

Salir

for

Se utiliza cuando se conoce la cantidad de repeticiones o cuando se recorre una coleccion, por ejemplo una lista de notas, productos o ejercicios.

notas = [8, 9, 7]
for nota in notas:
  print(nota)

while

Se utiliza cuando la repeticion depende de una condicion. Es util para validar datos, repetir menus o permitir varios intentos hasta que el usuario ingrese algo correcto.

intentos = 0
while intentos < 3:
  print("Intento", intentos + 1)
  intentos += 1

Importancia

Sin ciclos, un programador tendria que copiar y pegar instrucciones muchas veces. Con ciclos, el codigo se vuelve mas corto, ordenado y facil de mantener.

Contadores

Un contador es una variable que aumenta o disminuye para contar cuantas veces ocurre un evento.

Salir

Informacion ampliada

Los contadores se usan dentro de ciclos para registrar repeticiones, aciertos, errores, productos vendidos, estudiantes aprobados o intentos de acceso. Normalmente empiezan en cero y se incrementan con cada evento importante.

aprobados = 0
for nota in notas:
  if nota >= 7:
    aprobados = aprobados + 1

Acumuladores

Un acumulador guarda una suma progresiva de valores procesados dentro de un ciclo.

Salir

Informacion ampliada

A diferencia del contador, que registra cantidad de eventos, el acumulador suma valores. Es indispensable para calcular totales, promedios, ventas, consumos, sueldos o puntajes.

suma = 0
for nota in notas:
  suma = suma + nota
promedio = suma / len(notas)

Funciones

Una funcion agrupa instrucciones para reutilizarlas sin repetir codigo.

Salir

Informacion ampliada

Las funciones permiten dividir un programa en partes pequenas y faciles de probar. Reciben datos mediante parametros, ejecutan un proceso y pueden devolver un resultado con return. Gracias a ellas, el codigo se vuelve modular y mas ordenado.

def calcular_promedio(n1, n2, n3):
  return (n1 + n2 + n3) / 3

promedio = calcular_promedio(8, 9, 10)

Ventajas

  • Evitan repetir bloques de codigo.
  • Facilitan las pruebas y correcciones.
  • Permiten que cada parte del programa tenga una responsabilidad clara.

Logica de programacion

La logica permite pasar de una situacion confusa a una solucion clara, ordenada y comprobable.

Salir

Concepto ampliado

La logica de programacion es la capacidad de analizar un problema, separar sus partes importantes y construir una secuencia de instrucciones que produzca un resultado esperado. Antes de escribir codigo, el estudiante debe reconocer que datos recibe, que condiciones influyen, que calculos se hacen y que salida se debe entregar.

En un aplicativo educativo, la logica se demuestra cuando cada ejercicio tiene validacion, decision y retroalimentacion. Por ejemplo, no basta con calcular un promedio; el sistema debe decidir si el estudiante aprueba, explicar la formula y advertir si las notas ingresadas no son validas.

Elementos clave

  • Orden: las instrucciones se ejecutan en una secuencia comprensible.
  • Precision: cada paso evita ambiguedades y datos innecesarios.
  • Finitud: el procedimiento termina y entrega una respuesta.
  • Verificacion: se prueban casos normales, limites y erroneos.

Ejemplo guiado

Problema: saber si una persona puede obtener descuento estudiantil en transporte.

Entrada: edad, condicion de estudiante y valor del pasaje.
Proceso: comprobar si es estudiante y si la edad esta dentro del rango permitido.
Salida: valor final del pasaje y mensaje de retroalimentacion.
SI estudiante = verdadero Y edad <= 25 ENTONCES descuento = 50%

Algoritmos y entrada-proceso-salida

Un algoritmo traduce una necesidad real a pasos ejecutables. La tecnica EPS ayuda a no perderse.

Salir

Definicion ampliada

Un algoritmo es una secuencia finita y ordenada de instrucciones que resuelve un problema. Para que sea util en programacion debe especificar datos de entrada, operaciones, decisiones, repeticiones y salida. Esta estructura evita que el estudiante programe por intuicion y lo obliga a razonar el procedimiento.

ComponentePregunta guiaEjemplo: sueldo con horas extra
EntradaQue datos necesita el sistema?Horas trabajadas, pago por hora, horas extra.
ProcesoQue calculos o decisiones se aplican?Pago normal + horas extra * recargo.
SalidaQue resultado se muestra?Sueldo final y explicacion del calculo.

Caso resuelto: descuento en tienda

Entrada: subtotal y tipo de cliente
Proceso: si cliente = frecuente y subtotal >= 50, aplicar 15%; si no, aplicar 5% sobre compras mayores a 100
Salida: total a pagar y ahorro generado

Este caso usa operadores relacionales, operadores logicos y condicionales anidados. Es un buen ejemplo porque conecta programacion con una situacion cotidiana.

Variables, operadores y estructuras de control

Son las herramientas que hacen que un algoritmo deje de ser texto y se convierta en programa.

Salir

Variables

Guardan datos como nombres, precios, edades, notas o estados logicos. Deben tener nombres claros para que el algoritmo se lea como una explicacion.

subtotal = 85.50
cliente_frecuente = True

Operadores

Permiten calcular y comparar: suma, resta, multiplicacion, division, mayor que, menor que, igualdad, Y, O y NO.

subtotal >= 50 AND cliente_frecuente == True

Condicionales

Permiten decidir. Son esenciales para clasificar datos, validar formularios y dar retroalimentacion personalizada.

if promedio >= 7:
  estado = "Aprobado"

Ciclos

Repiten instrucciones. Se usan para recorrer listas de ejercicios, sumar ventas, contar intentos o generar una tabla de verdad.

for nota in notas:
  suma = suma + nota

POO Basica

La programacion orientada a objetos organiza el software mediante clases, objetos, atributos y metodos.

Salir

Informacion ampliada

Una clase es un molde que define atributos y comportamientos. Un objeto es una instancia concreta creada a partir de esa clase. Este enfoque ayuda a representar elementos reales dentro del software, como estudiantes, cuentas, productos, ejercicios o vehiculos.

En proyectos educativos, la POO permite organizar mejor la informacion. Por ejemplo, un ejercicio puede tener titulo, dificultad, enunciado y un metodo para validar respuestas.

class Ejercicio:
  def __init__(self, titulo, dificultad, solucion):
    self.titulo = titulo
    self.dificultad = dificultad
    self.solucion = solucion

  def validar(self, respuesta):
    return respuesta == self.solucion

Conceptos clave

ConceptoDescripcion simple
ClaseMolde o plantilla.
ObjetoElemento creado desde una clase.
AtributoDato que pertenece al objeto.
MetodoAccion que puede realizar el objeto.
